Install guide · v1.2.1

安装 Ivyea Agent

一条命令装完，三步就能对话。安装脚本会自动检测 Python、装好 pipx、拉取最新 Release，把 ivyea 作为独立命令放进 PATH——不污染系统 Python，升级卸载都干净。Windows / macOS / Linux 三平台，CI 全绿。

最快安装

选择你的系统与方式，复制整段运行即可。一键脚本是最省心的方式；已经装好 pipx 的用 pipx 直装源码。

# 1 · 一键安装（自动装 pipx，拉最新 Release wheel）
$ curl -fsSL https://raw.githubusercontent.com/Hector-xue/ivyea-agent/main/scripts/install.sh | bash
[ivyea] 使用 Python 3.11.9
[ivyea] 安装 ivyea-agent（来源：Release wheel v1.2.1）…
[ivyea] ✓ 安装完成。
# 2 · 首次引导：选模型 → 配 key（可选接领星）
$ ivyea onboard
# 3 · 开始对话
$ ivyea chat
→ 对话式运营 Agent 就绪 ✓

看不到 ivyea 命令？多半是 PATH 还没刷新。重开一个终端即可；Linux/macOS 也可先临时执行 export PATH="$HOME/.local/bin:$PATH"。

一键脚本做了什么

脚本是幂等的——重复运行就是升级到最新版。全过程只做这四件事，任何一步失败都会给出明确提示：

01

检测 Python

找 Python 3.9+，缺失时按系统包管理器（apt/dnf/yum/apk/brew）尝试自动安装。

02

准备 pipx

没有 pipx 就装一个，并把 ~/.local/bin 加入 PATH，隔离运行环境。

03

拉 Release

从 GitHub Release 直链下载 wheel（绕开 API 限流）；不可用时回退 git 源码。

04

装好自检

pipx install 后跑 self doctor，并初始化本地检索索引。

系统要求

项目要求
Python3.9 或更高（脚本会自动检测，必要时尝试自动安装）。
系统Windows、macOS、Linux。CI 三平台 × Python 3.9 / 3.11 / 3.12 全绿。
网络在线安装需能访问 GitHub。国内可设 PIP_INDEX_URL 走镜像，私有仓库设 GITHUB_TOKEN；完全离线见「离线内网部署」。
模型DeepSeek、OpenAI 兼容、Anthropic、Gemini、OpenAI Codex 等任选其一作主脑，ivyea onboard 里配置。

固定版本安装

团队协作时建议固定版本，避免不同成员装到不同 Release。用 IVYEA_VERSION 指定标签即可。

固定 v1.2.1
# Linux / macOS
$ curl -fsSL https://raw.githubusercontent.com/Hector-xue/ivyea-agent/main/scripts/install.sh | IVYEA_VERSION=v1.2.1 bash
 
# Windows PowerShell
PS> $env:IVYEA_VERSION="v1.2.1"
PS> iwr https://raw.githubusercontent.com/Hector-xue/ivyea-agent/main/scripts/install.ps1 -UseBasicParsing | iex

pipx / 手动安装

不想跑脚本，或想直接从源码装，用 pipx 一行即可。开发者用 editable 模式跑本地改动。

pipx / venv
# A · pipx 从源码直装（推荐给已有 pipx 的人）
$ pipx install git+https://github.com/Hector-xue/ivyea-agent.git
 
# B · 开发者：clone + editable，改代码即时生效
$ git clone https://github.com/Hector-xue/ivyea-agent.git
$ cd ivyea-agent
$ python -m venv .venv && . .venv/bin/activate
$ pip install -e ".[dev,anthropic]"
需要本地语义检索（sentence-transformers）时，一键脚本加 IVYEA_WITH_SEMANTIC=1，或 pipx 装完执行 pipx inject ivyea-agent "sentence-transformers>=3.0"。不装也能用，默认回退本地 hash 索引。

首次配置

ivyea onboard 是交互式引导：选主脑模型、填 API key、可选接入领星 OpenAPI、生成 AGENTS.md 持久指令。全程回车即用默认值。

ivyea onboard
$ ivyea onboard
选择主脑模型：
› deepseek · openai · anthropic · gemini · openai-codex …
填入 API key（留空可稍后 ivyea config 补）： ••••••••
接入领星 OpenAPI 广告数据？(y/N)： N
✓ 已写入 ~/.ivyea/settings.json 与 .env
✓ 已生成 AGENTS.md（持久运营指令）
下一步：ivyea chat

之后随时可用这些命令查看或修改配置：

ivyea config show            # 查看当前配置
ivyea config set key value   # 改单项
ivyea model                  # 查看 / 切换主脑模型（交互）
配置、记忆与知识库默认都在 ~/.ivyea/：settings.json、.env、mcp.json、memory.db 和用户知识库。备份这个目录就是备份全部状态。

验证与维护

ivyea doctor 一键体检：配置、依赖、知识库、磁盘、领星 / MCP 连接。装完跑一次确认无红灯。

ivyea doctor
$ ivyea doctor
✓ Python 3.11.9 · ivyea-agent 1.2.1
✓ 主脑模型已配置 · API key 就绪
✓ 知识库 · 本地检索索引已建
✓ 数据目录 ~/.ivyea 可写 · 磁盘空间充足
○ 领星 OpenAPI 未接入（可选，ivyea lingxing setup）

升级与卸载都走 ivyea self（默认 dry-run，加 --execute 才真正执行）：

ivyea self upgrade --execute              # 升级到最新 Release
ivyea self upgrade --version v1.2.1 --execute   # 升到指定版本
ivyea self uninstall --execute            # 卸载（保留数据）
ivyea self uninstall --remove-data --execute    # 卸载并删除 ~/.ivyea

重新跑一键安装脚本也会把已装的 ivyea 升级到最新版，效果等同 self upgrade。

离线内网部署

内网或无外网机器：在有网的机器上打一个离线包（含 wheelhouse 与启动器），复制进内网解压安装，全程不下载任何 Python 包。

离线包
# 有网机器：生成离线包
$ python scripts/build_offline_bundle.py
产物在 dist/offline/：
ivyea-agent-offline-1.2.1.tar.gz (Linux / macOS)
ivyea-agent-offline-1.2.1.zip (Windows)
 
# 内网机器：解压后安装（不联网）
$ bash install.sh # Windows: powershell -ExecutionPolicy Bypass -File .\install.ps1
[ivyea] ✓ 离线安装完成。
要把本地语义检索依赖或 embedding 模型也打进离线包，加 --with-semantic（可选 --semantic-model-dir 指定预下载的模型目录），详见 README。

故障排查

现象处理方式
装完找不到 ivyeaPATH 未刷新。重开终端；或执行 pipx ensurepath 后重开，Linux/macOS 也可 export PATH="$HOME/.local/bin:$PATH"。
Python 版本太低安装 Python 3.9+。Windows 从 python.org 安装时务必勾选 Add Python to PATH。
Release 读取失败 / 403多为 GitHub API 限流或私有仓库。设 GITHUB_TOKEN，或设 IVYEA_REF=main 从 git 源码安装。
pip 下载太慢设镜像：PIP_INDEX_URL=https://pypi.tuna.tsinghua.edu.cn/simple（放在安装命令前）。
Windows 报执行策略限制当前会话放行：Set-ExecutionPolicy -Scope Process Bypass，再运行安装命令。
中文 CSV 乱码优先用 .xlsx，或把 CSV 另存为 UTF-8 编码。
