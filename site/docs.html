Ivyea Agent 文档

Ivyea Agent 是本地优先的 Amazon 运营 CLI Agent。核心原则是：规则引擎先给确定性结论，LLM 负责解释、复核、补充知识和执行流程，但真实写入必须经过审批。

概览

模块能力
广告巡检搜索词报告分析、否词候选、降 bid、放量、Listing 反馈、人工复核。
对话 Agent自然语言触发巡检、知识检索、动作队列和工具调用。
Skills内置 Amazon 搜索词优化、否词风控、预算节奏、Listing 诊断、新品启动、周度复盘。
知识库官方/社区/用户知识卡，SQLite FTS5 索引，冲突审计。
写入执行人工审批、保护词、动作队列、审计日志、回滚。
视觉诊断图片资产检查、OCR、多模态模型请求包和可选真实调用。

安装

普通用户推荐一键脚本。更完整的离线、内网和企业部署见仓库内 `docs/部署指南.md`。

# Linux / macOS
curl -fsSL https://raw.githubusercontent.com/Hector-xue/ivyea-agent/main/scripts/install.sh | bash

# Windows PowerShell
iwr https://raw.githubusercontent.com/Hector-xue/ivyea-agent/main/scripts/install.ps1 -UseBasicParsing | iex

安装固定版本：

curl -fsSL https://raw.githubusercontent.com/Hector-xue/ivyea-agent/main/scripts/install.sh | IVYEA_VERSION=v0.5.2 bash

配置

ivyea config
ivyea config show
ivyea model list
ivyea model deepseek-chat

配置文件都存放在本机 `~/.ivyea/`。API key 写入 `.env`，模型、站点和默认 ACOS 写入 `settings.json`。

运营能力

广告巡检

ivyea patrol 搜索词报告.xlsx --asin B0XXXXXXXX --site US --target-acos 0.3
ivyea patrol 搜索词报告.csv --no-llm

输出 Markdown 报告，包含动作、证据、风险和置信度。无模型 key 时会自动降级为规则引擎。

Listing / Review / Offer / 竞品诊断

ivyea listing audit --title "Karaoke Machine" --search-terms "karaoke machine kids,studio mic"
ivyea review audit --reviews "质量差，不能用，少件" --rating 3.7 --review-count 4
ivyea offer audit --margin-rate 0.3 --target-acos 0.35 --inventory-days 9
ivyea competitor audit --own-terms "karaoke machine" --search-terms "acme karaoke,B0ABCDEFGH"

这些命令用于判断广告表现差是流量问题、Listing 承接问题、评论信任问题、价格/库存问题还是竞品串号问题。

Skills / 知识库

ivyea skill list
ivyea skill search 否词
ivyea skill show amazon.search_term_optimizer
ivyea knowledge audit
ivyea knowledge import ./note.md --id user.playbook --tags "listing,预算"
ivyea knowledge index
ivyea knowledge conflicts

用户知识库存放在 `~/.ivyea/knowledge/`。导入时会记录来源类型、可信度、更新时间、URL、license 和正文 hash。

审批执行

ivyea apply 巡检输出目录
ivyea apply 巡检输出目录 --protected "品牌词,核心词"
ivyea apply 巡检输出目录 --execute --from-mcp writer
ivyea audit list
ivyea audit rollback <审计ID>

默认 dry-run。真实写入需要显式 `--execute`，并且 MCP 服务器必须配置 `writeActions`。品牌词、核心词、保护词、低置信度动作会被护栏拦截。

MCP

ivyea mcp add
ivyea mcp tools <名称>
ivyea mcp call <名称> <工具> --args '{"asin":"B0..."}'
ivyea mcp suggest <名称> <工具> --args '{"asin":"{asin}","site":"{site}","days":"{days}"}'
ivyea mcp validate <名称>

MCP 客户端是通用映射模式，不绑定厂商字段。`mcp suggest` 会根据工具返回自动推断 `dataSource.rows_path` 和搜索词报告字段映射。

图片视觉

ivyea image audit ./listing-images --prompt
ivyea image ocr ./listing-images --lang eng+chi_sim
ivyea image vision ./listing-images --provider openai --payload --output reports/openai-vision.json
OPENAI_API_KEY=... ivyea image vision ./listing-images --provider openai --call

`image vision` 默认只生成请求包；传 `--call` 才会真实调用 OpenAI、Anthropic 或 Gemini 多模态 API。

自动化

ivyea weekly review --output reports/weekly.md
ivyea alert check
ivyea alert check --notify --channel feishu
ivyea schedule set daily-alert alert --every-hours 24 --notify --channel webhook
ivyea schedule run-due
ivyea eval

`schedule` 不是常驻服务，适合由 cron、systemd timer、Windows Task Scheduler 或 IvyeaOps 定时调用。

安全

  • API key、token、secret 在工具输出、trace 和通知中会脱敏。
  • 写文件、执行命令、真实广告写入都有审批或策略限制。
  • `~/.ivyea/policy.json` 可限制读写目录和命令 allow/deny。
  • 真实写入有审计记录和回滚入口。
ivyea policy init
ivyea policy show
ivyea policy check-command "git reset --hard"

门户部署

本网站源码在 `site/`。正式部署和 `ivyea.com` 一样，静态文件放到 `/var/www/agent.ivyea.com/`，Nginx 读取该目录。

rsync -a --delete site/ /var/www/agent.ivyea.com/
cp deploy/nginx/agent.ivyea.com.conf /etc/nginx/conf.d/agent.ivyea.com.conf
nginx -t && systemctl reload nginx
certbot --nginx -d agent.ivyea.com

DNS 需要把 `agent.ivyea.com` 的 A 记录指向当前服务器公网 IP。
