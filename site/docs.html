文档中心

这份文档按实际使用路径组织：先安装和配置，再跑 CSV 或领星广告巡检，随后接入知识库、MCP、审批写入、图片视觉和自动化任务。

产品概览

核心原则

规则引擎先给确定性结论，LLM 负责解释、复核和流程编排；真实写入必须经过人工审批、护栏校验和审计记录。

模块解决的问题常用命令
广告巡检识别浪费词、放量词、降 bid 候选和 Listing 反馈。ivyea patrol
领星链路按店铺拉取真实广告数据，带缓存、operate 总闸和审批写入。ivyea lingxing
对话 Agent自然语言触发巡检、工具调用和任务计划。ivyea chat
知识库沉淀官方知识、社区经验和团队打法。ivyea knowledge
写入执行审批式落地否词、出价和预算动作。ivyea apply
视觉诊断检查 Listing 图片质量、文字和合规风险。ivyea image

快速开始

ivyea onboard
ivyea patrol 搜索词报告.xlsx --asin B0XXXXXXXX --site US --target-acos 0.3
ivyea chat

未配置模型 key 时，广告巡检会自动降级为规则引擎输出，不会中断。

运营命令

广告搜索词巡检

ivyea patrol 搜索词报告.xlsx --asin B0XXXXXXXX --site US --target-acos 0.3
ivyea patrol 搜索词报告.csv --no-llm

CSV/XLSX 模式适合没有 API 凭据的新用户，也适合把 Amazon Ads 后台导出的搜索词报告拿来离线分析。

领星 OpenAPI 店铺巡检

ivyea lingxing setup
ivyea lingxing probe
ivyea lingxing sellers
ivyea patrol --from-lingxing --sid 1863 --days 30
ivyea shadow off
ivyea lingxing operate on
ivyea patrol --from-lingxing --sid 1863 --days 30 --execute

领星链路是当前推荐的真实广告数据入口。只读巡检会使用本地缓存和逐日进度；真实写入必须关闭影子模式、打开 operate 总闸，并逐条人工审批。

Listing / Review / Offer / 竞品

ivyea listing audit --title "Karaoke Machine" --search-terms "karaoke machine kids,studio mic"
ivyea review audit --reviews "质量差，不能用，少件" --rating 3.7 --review-count 4
ivyea offer audit --margin-rate 0.3 --target-acos 0.35 --inventory-days 9
ivyea competitor audit --own-terms "karaoke machine" --search-terms "acme karaoke,B0ABCDEFGH"

知识库与 Skills

ivyea skill list
ivyea skill search 否词
ivyea skill show amazon.search_term_optimizer
ivyea knowledge audit
ivyea knowledge import ./note.md --id user.playbook --tags "listing,预算"
ivyea knowledge index
ivyea knowledge conflicts

用户知识库存放在 ~/.ivyea/knowledge/。导入时记录来源类型、可信度、更新时间、URL、license 和正文 hash。

审批执行

ivyea apply 巡检输出目录
ivyea apply 巡检输出目录 --protected "品牌词,核心词"
ivyea apply 巡检输出目录 --execute --from-mcp writer
ivyea audit list
ivyea audit rollback <审计ID>

默认 dry-run。领星写入需要 ivyea lingxing operate on；通用 MCP 写入需要配置 writeActions。两种路径都会走审批和审计。

MCP 接入

ivyea mcp add
ivyea mcp tools <名称>
ivyea mcp call <名称> <工具> --args '{"asin":"B0..."}'
ivyea mcp suggest <名称> <工具> --args '{"asin":"{asin}","site":"{site}","days":"{days}"}'
ivyea mcp validate <名称>

MCP 是通用数据源和写入映射，不等于领星广告源。领星广告数据请优先走 ivyea lingxing；MCP 适合 Sorftime、SIF、自建服务或其它内部数据接口。

图片视觉

ivyea image audit ./listing-images --prompt
ivyea image ocr ./listing-images --lang eng+chi_sim
ivyea image vision ./listing-images --provider openai --payload --output reports/openai-vision.json
OPENAI_API_KEY=... ivyea image vision ./listing-images --provider openai --call

image vision 默认只生成请求包；传 --call 才会真实调用多模态 API。

自动化

ivyea weekly review --output reports/weekly.md
ivyea alert check
ivyea alert check --notify --channel feishu
ivyea schedule set daily-alert alert --every-hours 24 --notify --channel webhook
ivyea schedule run-due
ivyea eval

schedule 不是常驻服务，适合由 cron、systemd timer、Windows Task Scheduler 或 IvyeaOps 定时调用。

安全策略

  • API key、token、secret 在工具输出、trace 和通知中自动脱敏。
  • 写文件、执行命令和真实广告写入都有审批或策略限制。
  • ~/.ivyea/policy.json 可限制读写目录和命令 allow/deny。
  • 真实写入有审计记录和回滚入口。
ivyea policy init
ivyea policy show
ivyea policy check-command "git reset --hard"

门户部署

本网站源码在 site/。正式部署和 ivyea.com 一样，静态文件放到 /var/www/agent.ivyea.com/，Nginx 读取该目录。

mkdir -p /var/www/agent.ivyea.com
cp -a site/. /var/www/agent.ivyea.com/
cp deploy/nginx/agent.ivyea.com.conf /etc/nginx/conf.d/agent.ivyea.com.conf
nginx -t && nginx -s reload
certbot --nginx -d agent.ivyea.com
