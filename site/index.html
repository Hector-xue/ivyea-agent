为什么要单独做一个 Amazon Agent

通用编码 Agent 解决不了广告报表、否词误伤、Listing 承接、预算节奏和真实写入审批这些运营问题。Ivyea Agent 把这些流程产品化。

广告巡检

读取搜索词报告，输出否词、放量、降 bid、Listing 反馈和人工复核建议，每条建议都有证据。

知识库和 Skills

内置 Amazon Ads、Listing、预算、Launch、周度复盘等运营流程，支持导入团队知识。

审批式执行

真实写入前逐条确认，保护品牌词/核心词，记录审计日志并支持回滚。

视觉诊断

本地图片尺寸检查、OCR、多模态模型审核请求包，以及可选真实调用。

自动预警

检查动作积压、工具失败、知识库缺失、画像配置和磁盘空间，可发 webhook 或飞书通知。

本地私有

配置、API key、记忆、知识库默认都在本机 `~/.ivyea/`，不依赖外部数据库。

一条命令安装

普通用户不需要 clone 仓库。脚本会安装 pipx，并优先从 GitHub Release wheel 安装最新稳定版本。

Linux / macOS

curl -fsSL https://raw.githubusercontent.com/Hector-xue/ivyea-agent/main/scripts/install.sh | bash

Windows PowerShell

iwr https://raw.githubusercontent.com/Hector-xue/ivyea-agent/main/scripts/install.ps1 -UseBasicParsing | iex

固定版本

curl -fsSL https://raw.githubusercontent.com/Hector-xue/ivyea-agent/main/scripts/install.sh | IVYEA_VERSION=v0.5.2 bash

部署门户网站

`site/` 是静态站点，正式部署方式和 ivyea.com 一样：文件同步到服务器 `/var/www/agent.ivyea.com/`，由 Nginx 对外提供服务。

1
同步 site/

执行 `rsync -a --delete site/ /var/www/agent.ivyea.com/`。

2
配置 Nginx

使用 `deploy/nginx/agent.ivyea.com.conf`，风格和 `ivyea.com.conf` 一致。

3
开启 HTTPS

DNS 指向服务器后执行 `certbot --nginx -d agent.ivyea.com`。
