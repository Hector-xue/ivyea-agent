# Seller returns and SAFE-T claim evidence

Source basis: Amazon SP-API Returns Reports documentation. Eligibility, filing windows, refund workflows, and claim outcomes are dynamic Seller Central rules.

## Official report facts

- Amazon returns reports can expose return-request dates/status, RMA identifiers, label details, reason codes, resolution, refund amount, and delivery information.
- Current flat-file return reports may also expose SAFE-T action reason, claim identifier, claim state, creation time, reimbursement amount, and refunded amount.
- Report type, availability, roles, format, date-range limit, and marketplace behavior must be verified against current documentation.

## Claim evidence packet

Preserve:

- order, ASIN/SKU, quantity, RMA, tracking, carrier, delivery and refund timestamps;
- return reason and condition, package/serial-number evidence, inspection photos or video, and discrepancy notes;
- buyer communication, refund action, label payer/type, return policy state, and any Amazon notification;
- claim identifier, reason selected, submission timestamp, decision, reimbursement, and appeal correspondence.

## Decision boundaries

- Do not infer SAFE-T eligibility or a filing deadline from an old guide or another marketplace; verify the current authenticated workflow.
- A return reason code is a recorded selection, not proof of product condition or fault.
- Separate refund, return, A-to-z claim, chargeback, SAFE-T claim, and FBA reimbursement processes.
- Never fabricate condition evidence, backdate a claim, or present an expected reimbursement as approved.
