# Launch versus mature ASIN advertising strategy

## Why this card exists

Amazon's sponsored ads guidance supports using Sponsored Products early, using
automatic targeting to learn shopping queries, and then using search term data
to build manual keyword targeting. Operators should not apply mature-ASIN cost
rules to a launch ASIN without checking the lifecycle stage.

## Launch ASIN mode

- Goal: collect query data, validate relevance, and build conversion signals.
- Use automatic targeting and broad/phrase discovery with controlled budgets.
- Do not aggressively negate strategic category terms only because early ACoS is high.
- Treat high-click zero-order terms as investigation candidates first: check price,
  reviews, coupon, image clarity, offer, and listing term alignment.
- Graduate winners into manual phrase/exact campaigns after enough orders or clear
  conversion evidence.
- Keep budget large enough to generate signal, but isolate discovery budgets from
  core exact budgets.

## Mature ASIN mode

- Goal: protect profitable demand, scale proven winners, and remove waste.
- Enforce target ACoS more strictly for non-strategic terms.
- Separate harvesting, ranking-defense, competitor, and research traffic where possible.
- Use search term, targeting, placement, and performance-over-time reports together.
- Make bid changes in small increments and keep a stability window before judging.

## Conflict rule

If the profile says `stage=launch`, prioritize learning and relevance checks before
hard suppression. If the profile says `stage=mature`, prioritize efficiency and
budget reallocation unless a term is marked as core or protected.
