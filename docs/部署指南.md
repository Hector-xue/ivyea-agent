# Ivyea Agent 部署指南

本文面向三类人：

- 普通运营用户：只想在自己的 Windows / macOS / Linux 电脑上运行 `ivyea`。
- 团队管理员：要给团队准备统一安装方式、代理、镜像或内网安装包。
- 服务器运维：要把文档门户 `agent.ivyea.com` 部署出来。

## 推荐方案

| 场景 | 推荐方式 | 说明 |
| --- | --- | --- |
| 个人电脑 | 一键安装脚本 | 自动安装 pipx，再安装最新 Release wheel |
| 多台电脑 | 固定版本一键安装 | 设置 `IVYEA_VERSION=v0.5.3`，保证全员版本一致 |
| 内网/离线 | 离线 bundle | 先生成带 `wheelhouse/` 的压缩包，再复制到内网 |
| 开发者 | `pipx install ./ivyea-agent` 或 `pip install -e .` | 适合修改源码 |
| 门户网站 | Nginx 静态站 | 和 `ivyea.com` 一样部署到本机 `/var/www/agent.ivyea.com` |

## 一键安装

### Linux / macOS

```bash
curl -fsSL https://raw.githubusercontent.com/Hector-xue/ivyea-agent/main/scripts/install.sh | bash
```

固定版本：

```bash
curl -fsSL https://raw.githubusercontent.com/Hector-xue/ivyea-agent/main/scripts/install.sh | IVYEA_VERSION=v0.5.3 bash
```

国内镜像：

```bash
curl -fsSL https://raw.githubusercontent.com/Hector-xue/ivyea-agent/main/scripts/install.sh | \
  PIP_INDEX_URL=https://pypi.tuna.tsinghua.edu.cn/simple bash
```

### Windows PowerShell

```powershell
iwr https://raw.githubusercontent.com/Hector-xue/ivyea-agent/main/scripts/install.ps1 -UseBasicParsing | iex
```

固定版本：

```powershell
$env:IVYEA_VERSION="v0.5.3"
iwr https://raw.githubusercontent.com/Hector-xue/ivyea-agent/main/scripts/install.ps1 -UseBasicParsing | iex
```

## 安装脚本行为

脚本会按顺序做这些事：

1. 检查 Python 版本，要求 Python 3.9+。
2. 如果没有 Python，尽量自动安装：
   - Linux：`apt-get` / `dnf` / `yum` / `apk`
   - macOS：Homebrew
   - Windows：winget
3. 如果脚本旁边有 `wheelhouse/`，优先使用本机 venv 离线安装，不依赖 pipx 联网初始化。
4. 在线模式下，如果没有 pipx，自动安装 pipx。
5. 读取 GitHub 最新 Release，选择 `.whl` 安装。
6. 如果 Release 不可用，回退到 `git+https://github.com/Hector-xue/ivyea-agent.git@main`。
7. 安装完成后提示运行 `ivyea config`。

可用环境变量：

| 变量 | 示例 | 作用 |
| --- | --- | --- |
| `IVYEA_VERSION` | `latest` / `v0.5.3` | 指定 Release 版本 |
| `IVYEA_REF` | `main` / `v0.5.3` | 强制从 git 分支或 tag 安装 |
| `IVYEA_LOCAL` | `/opt/ivyea-agent` | 从本地源码安装 |
| `IVYEA_GITHUB_REPO` | `Hector-xue/ivyea-agent` | 覆盖 GitHub 仓库 |
| `GITHUB_TOKEN` | `ghp_...` | 私有仓库读取 Release 资产 |
| `PIP_INDEX_URL` | 清华源 URL | pip 依赖下载镜像 |
| `IVYEA_AUTO_INSTALL` | `0` / `1` | 是否允许脚本自动安装 Python/pipx |
| `IVYEA_WHEELHOUSE` | `/mnt/share/wheelhouse` | 指定离线依赖目录 |

## 手动安装

### pipx 从 Release wheel 安装

从 GitHub Release 下载 wheel 后：

```bash
pipx install ./ivyea_agent-0.5.3-py3-none-any.whl
```

### pipx 从源码安装

```bash
git clone https://github.com/Hector-xue/ivyea-agent.git
pipx install ./ivyea-agent
```

### 开发模式

```bash
git clone https://github.com/Hector-xue/ivyea-agent.git
cd ivyea-agent
python -m venv .venv
. .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -e ".[dev,anthropic]"
python -m pytest
```

## 首次配置

安装后执行：

```bash
ivyea config
```

配置文件位置：

| 文件 | 用途 |
| --- | --- |
| `~/.ivyea/settings.json` | 模型、站点、默认 ACOS 等 |
| `~/.ivyea/.env` | API key，本机保存 |
| `~/.ivyea/mcp.json` | MCP 服务器、数据源、写入映射 |
| `~/.ivyea/memory.db` | 记忆数据库 |
| `~/.ivyea/knowledge/` | 用户知识库 |

## 升级

一键脚本可重复执行，默认升级到最新 Release：

```bash
curl -fsSL https://raw.githubusercontent.com/Hector-xue/ivyea-agent/main/scripts/install.sh | bash
```

pipx 手动升级：

```bash
pipx upgrade ivyea-agent
```

如果是本地源码安装：

```bash
cd ivyea-agent
git pull
pipx reinstall ./ivyea-agent
```

## 卸载

```bash
pipx uninstall ivyea-agent
```

卸载不会删除 `~/.ivyea/`。如果要清空本地配置和记忆，再手动删除：

```bash
rm -rf ~/.ivyea
```

Windows：

```powershell
Remove-Item -Recurse -Force $HOME\.ivyea
```

## 离线/内网部署（推荐）

在有外网的发版机上生成完整离线包：

```bash
python scripts/build_offline_bundle.py
```

生成：

```text
dist/offline/ivyea-agent-offline-0.5.3.zip
dist/offline/ivyea-agent-offline-0.5.3.tar.gz
dist/offline/ivyea-agent-offline-0.5.3/
```

包内结构：

```text
install.sh
install.ps1
wheelhouse/
  ivyea_agent-0.5.3-py3-none-any.whl
  pandas-...
  openpyxl-...
  httpx-...
  prompt_toolkit-...
  cryptography-...
```

用户解压后：

Linux / macOS：

```bash
bash install.sh
```

Windows PowerShell：

```powershell
powershell -ExecutionPolicy Bypass -File .\install.ps1
```

脚本发现同目录 `wheelhouse/` 后，会自动使用：

```bash
python -m venv ~/.ivyea/runtime
~/.ivyea/runtime/bin/pip install --no-index --find-links ./wheelhouse ./wheelhouse/ivyea_agent-*.whl
```

并生成 `~/.local/bin/ivyea` 启动器。Windows 会安装到 `~\.ivyea\runtime` 并生成 `~\.ivyea\bin\ivyea.cmd`。这样安装 Ivyea Agent 和 Python 包依赖时不访问外网，也避开 pipx 首次初始化时尝试联网升级 pip 的问题。

注意：如果用户机器完全没有 Python 3.9+，纯离线包无法凭空安装 Python；这部分需要企业管理员提前通过系统镜像、软件中心、winget 缓存包或 Python 官方安装包准备好。

## 手工离线安装

1. 在有外网的机器下载 Release wheel。
2. 下载依赖 wheel：

```bash
mkdir wheelhouse
pip download ivyea_agent-0.5.3-py3-none-any.whl -d wheelhouse
```

3. 复制 `wheelhouse/` 到内网机器。
4. 内网安装：

```bash
pipx install --pip-args="--no-index --find-links ./wheelhouse" ./wheelhouse/ivyea_agent-0.5.3-py3-none-any.whl
```

## 企业统一部署建议

- 固定 `IVYEA_VERSION`，避免全员自动漂移到新版本。
- 使用企业 pip 镜像缓存依赖。
- 使用统一的 `~/.ivyea/policy.json` 模板限制命令和文件目录。
- 对真实写入链路启用 `operate` 开关、审批和审计回滚。
- 先在 dry-run 环境验证，再开放 `--execute`。

## 门户网站 agent.ivyea.com

仓库内 `site/` 是静态网站。正式部署方式和 `ivyea.com` 一样：文件放到服务器 `/var/www/agent.ivyea.com/`，Nginx 读取该目录，对外提供 `agent.ivyea.com`。

### 服务器目录

```bash
mkdir -p /var/www/agent.ivyea.com
rsync -a --delete site/ /var/www/agent.ivyea.com/
```

### Nginx

仓库提供配置模板：

```bash
cp deploy/nginx/agent.ivyea.com.conf /etc/nginx/conf.d/agent.ivyea.com.conf
nginx -t
systemctl reload nginx
```

配置风格和 `/etc/nginx/conf.d/ivyea.com.conf` 一致：静态目录、`try_files`、`/.well-known/acme-challenge/`、静态资源缓存和 `index.html` no-cache。

### DNS

在 DNS 服务商添加 A 记录，指向当前服务器公网 IP：

```text
agent.ivyea.com A <服务器公网 IP>
```

如果当前 `ivyea.com` 是 Cloudflare 托管，也在 Cloudflare 里加同样的 `agent` 记录即可。是否开启橙云按现有 `ivyea.com` 策略处理。

### HTTPS

DNS 生效后运行：

```bash
certbot --nginx -d agent.ivyea.com
```

生成证书后，certbot 会自动把 443 server 写入 Nginx 配置。再次执行：

```bash
nginx -t
systemctl reload nginx
```

## 故障排查

### 找不到 ivyea 命令

Linux/macOS：

```bash
pipx ensurepath
export PATH="$HOME/.local/bin:$PATH"
```

Windows：重开 PowerShell；如果仍失败，执行：

```powershell
py -m pipx ensurepath
```

### Python 版本太低

安装 Python 3.9+。Windows 安装时勾选 “Add Python to PATH”。

### GitHub Release 读取失败

- 公共仓库一般可直接读取。
- 私有仓库需要设置 `GITHUB_TOKEN`。
- 也可设置 `IVYEA_REF=main` 从 git 安装。

### 中文 CSV 乱码

优先使用 `.xlsx`，或把 CSV 另存为 UTF-8。

### 真实写入失败

检查：

```bash
ivyea doctor
ivyea mcp validate <名称>
ivyea audit list
```

真实执行需要 `--execute --from-mcp <名称>`，且 MCP 配好 `writeActions`。
