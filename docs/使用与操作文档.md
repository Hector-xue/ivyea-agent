# Ivyea Agent 使用与操作文档

> 自托管的**亚马逊运营对话式 Agent**。专长广告巡检（确定性规则引擎 + LLM 复核 + 护栏 + 审核制写入 + 审计回滚），也能干日常运营杂活（读 Excel、算数、写报告、查网页）。
> 哲学：**证据驱动、写操作一律人工审批、数据私有**。

---

## 一、安装

```bash
# 方式 A：装 Release 里的 wheel（任何人可用）
pip install https://github.com/Hector-xue/ivyea-agent/releases/download/v0.4.0/ivyea_agent-0.4.0-py3-none-any.whl

# 方式 B：从源码（pipx，装完任意目录敲 ivyea）
pipx install git+https://github.com/Hector-xue/ivyea-agent.git

# 方式 C：克隆后开发安装
git clone https://github.com/Hector-xue/ivyea-agent.git && cd ivyea-agent
pip install -e ".[anthropic]"     # anthropic 可选(用 Claude 才需要)
```
跨平台纯 Python（≥3.9），Windows/macOS/Linux 均测过。

## 二、首次运行（引导）

直接敲 `ivyea`，**首次会自动进引导**（也可手动 `ivyea onboard`）：

1. **选主脑模型并配 key** —— 推荐 DeepSeek（便宜、工具调用稳）；也可 Claude（Opus/Sonnet/Haiku，需 `ANTHROPIC_API_KEY`）或任意 OpenAI 兼容端点。
2. **（可选）接领星 OpenAPI** —— 拉真实广告数据 / 写回。填 host / appId / appSecret。
3. **（可选）生成 `AGENTS.md`** —— 写你的长期打法/边界，对话时自动注入。

配置都存在本机 `~/.ivyea/`（`.env` 密钥 600 权限，`settings.json`，`mcp.json`）。

## 三、两种用法

### 1) 对话模式（推荐）
```bash
ivyea                      # 进对话；裸敲即可
ivyea --continue           # 续接上次会话
ivyea chat --resume <id>   # 续接指定会话
ivyea --raw                # 原始流式输出（默认 Markdown 渲染）
```
直接说人话，它自己编排工具：「看下 sid 1863 这周广告」「读 /path/report.csv 帮我分析」。写操作一律弹**人工审批**。

**斜杠命令**：
| 命令 | 作用 |
|---|---|
| `/help` | 命令与用法 |
| `/model` `/config` | 切模型/配 key、配置向导 |
| `/plan` `/approve` | 进/出**计划模式**（只读，不写）；批准后执行 |
| `/cost` | 本会话 + 今日 token 用量与成本估算 |
| `/compact` | 压缩上下文（LLM 摘要历史，省 token，摘要入库） |
| `/init` | 生成账户指令模板 `AGENTS.md` |
| `/memory` | 记忆状态 / `/memory <词>` 检索 |
| `/tools` `/mcp` | 列工具 / MCP 服务器 |
| `/raw` `/clear` `/exit` | 切渲染 / 清上下文 / 退出 |

对话中可直接说「**记住**…」沉淀打法、「**回忆**…」跨会话召回。

### 2) 命令式（脚本/自动化）
```bash
# 领星店铺维度巡检（真实广告数据）
ivyea lingxing setup                       # 配凭据
ivyea lingxing probe                       # 自检（取令牌+店铺）
ivyea lingxing sellers                     # 列店铺拿 SID
ivyea patrol --from-lingxing --sid 1863 --days 30          # 只读巡检（带缓存+进度条）
ivyea patrol --from-lingxing --sid 1863 --days 30 --execute # 巡检 + 逐条人工审批写入

# CSV 巡检（无领星凭据也能用）
ivyea patrol 搜索词报告.csv --asin B0XXXXXXXX --site US
ivyea apply 巡检输出目录 --execute --from-mcp 领星          # 审核制执行(MCP 写映射)

# 审计/回滚
ivyea audit list
ivyea audit rollback <审计ID>
```

## 四、安全护栏（动钱的底线）

- **写操作永远人工审批**：多档选项 `[1]是 [2]本会话都允许此类 [3]否 [4]改一下 [5]全部停`。
- **领星写总闸 operate**：`ivyea lingxing operate on|off|status`，**默认关**（关时只 dry-run 预览），开了也带 120 分钟自动过期。
- **硬护栏**（代码强制，LLM 绕不过）：否词 ≥15 点击 0 单才否；单次调 bid ≤±20%、步长 ≤15%；冷却 7 天；不否品牌词/竞品词/核心品类词/保护词；不投 SBV / 不操控评论；写失败自动**熔断**关开关。
- **记忆护栏**：你否过的词，下次巡检自动拦截、不再反复建议。
- **成本护栏**：`ivyea config set daily_cost_limit_cny 20` 设每日上限，超了暂停问你。
- **回滚**：每次写入留审计快照，`ivyea audit rollback <ID>` 一键回滚（否词→归档、调 bid/预算→还原旧值）。
- **计划模式 / 影子模式**：见下。

## 五、影子模式（重点）——动钱前先用数据换信任

> **这是 Ivyea Agent 独有、Claude Code/Codex/Hermes 都没有的能力。**
> 一个会动你广告预算的 Agent，最大的门槛是「我凭什么信它」。影子模式解决这个：
> **它先只看不写、把每条建议记下来，过几天用真实花费回测「如果当时照做，能省多少 / 多抓多少单」**——用你自己账户的真实数据证明它值得信，你看准了再放它真动手。

### 5.1 它怎么工作

1. **记账**：每次 `ivyea patrol --from-lingxing` 巡检，都会把候选建议（否词/收割/降bid/加bid/加预算）连同**当时的触发指标**记进影子台账（`~/.ivyea/shadow.db`）。同一店铺+杠杆+词 7 天内不重复记。
2. **只看不写**：开了影子模式后，即使你 `--execute`、即使 operate 开关是开的，**也绝不写广告**——只记录、不执行。这是比 operate 更强的一层「绝不动手」保险。
3. **回测**：过几天再 `ivyea shadow report`，它会**拉这几天的真实搜索词数据**，和当初记的建议对账：
   - **否词**：当初建议否的词，之后**还烧了多少钱、且仍 0 单** → 这就是「如果当时照做能省下的钱」。
   - **收割**：当初建议升精准的词，之后**又出了多少单** → 这就是「如果当时照做能抓住的增量」。
   - **诚实**：当初建议否、但后来**真转化出单**的词，**不计入节省**（不夸大、不给自己邀功）。

### 5.2 命令

```bash
ivyea shadow on                       # 开影子模式：巡检只记不写
ivyea shadow off                      # 关：恢复正常审批写入（仍需 operate 开关）
ivyea shadow list [--sid 1863]        # 看台账里记了哪些建议
ivyea shadow report --sid 1863 [--days 14]   # 拉近 N 天现况回测，出信任报告
```

### 5.3 推荐工作流（新店/新用户）

```
第 1 步  ivyea shadow on                              # 进影子模式
第 2 步  ivyea patrol --from-lingxing --sid 1863      # 跑巡检(只记不写)，隔几天跑一次，攒台账
   …    （这期间它一个广告都不会动）
第 3 步  ivyea shadow report --sid 1863               # 过 1~2 周，看"若照做的收益"
第 4 步  信任建立后：
        ivyea shadow off                              # 关影子模式
        ivyea lingxing operate on                     # 开领星写总闸
        ivyea patrol --from-lingxing --sid 1863 --execute   # 逐条人工审批，真动手
```

### 5.4 信任报告长什么样

```
# 影子模式信任报告 — sid 1876
> 回测了 4 条有后续数据的建议。
- 否词：2 个词若当时照做，至今已省 ¥32.50（之后仍纯烧钱0单）。
- 收割：1 个词若当时升精准，至今多抓 9 单。
  · 否词「karaoke remote」之后又烧 ¥21.50(0单) → 若照做已省
  · 否词「cheap mic」之后又烧 ¥11.00(0单) → 若照做已省
  · 收割「studio mic pro」之后又出 9 单 → 若照做已抓住
```
> 注意：上例里第 3 个被建议否的词后来转化出单了，报告**没有**把它算进节省——这种「不替自己邀功」的诚实，正是信任的来源。

### 5.5 影子模式 vs 计划模式 vs operate 开关（别混）

| 机制 | 作用范围 | 行为 |
|---|---|---|
| **计划模式** `/plan` | 单次对话 | 当轮只读、不调写工具；`/approve` 后才执行。临时「先别动手，给我看计划」 |
| **影子模式** `shadow on` | 持续，跨会话 | 巡检只记不写 + 回测换信任。长期的「先观察攒数据」 |
| **operate 开关** `lingxing operate on` | 领星写入总闸 | 关=dry-run，开=允许真写（仍需逐条审批）。影子模式开时，operate 即使开也不写 |

**优先级**：影子模式开 > operate 开关。只要影子模式开着，就绝不写。

## 六、记忆（越用越懂你的店）

- **持久指令**：`~/.ivyea/USER.md`（运营画像）+ `AGENTS.md`（账户打法/边界）+ 项目目录 `./AGENTS.md`，启动**自动注入**对话（CLAUDE.md/AGENTS.md 同款）。`/init` 或 onboard 生成模板。
- **会话回忆**：每轮对话入库，`ivyea memory search <词>` 或对话里「回忆…」跨会话召回。
- **决策记忆**：你的每次否决/批准都记下来 → 尊重历史否决、5 天稳定期不重复调。
- 查看：`ivyea memory`（状态/最近巡检）、`ivyea memory note B0XXX`（某 ASIN 笔记）。

## 七、模型与健壮性

- **多模型**：`ivyea model` 选 DeepSeek（主）/ Claude 原生（高端档，含 prompt caching）/ 通义/Kimi/GLM/豆包/OpenRouter/自定义。
- **降级链**：`ivyea config set fallback_models deepseek-reasoner,claude-sonnet` —— 主脑限流/挂了自动切有 key 的备用模型继续。
- **重试/退避**：429/5xx/网络错误自动重试 + 指数退避；4xx 直接报错。
- **可中断**：对话里 Ctrl+C 停本轮、不杀会话。

## 八、常见操作速查

| 我想… | 命令 |
|---|---|
| 第一次配置 | `ivyea`（自动引导）或 `ivyea onboard` |
| 换模型/配 key | `ivyea model` |
| 看店铺 SID | `ivyea lingxing sellers` |
| 只读巡检 | `ivyea patrol --from-lingxing --sid <SID> --days 30` |
| 先观察攒信任 | `ivyea shadow on` → 多次巡检 → `ivyea shadow report --sid <SID>` |
| 真动手改广告 | `ivyea shadow off` → `ivyea lingxing operate on` → `ivyea patrol … --execute` |
| 回滚某次写入 | `ivyea audit rollback <ID>` |
| 设每日花费上限 | `ivyea config set daily_cost_limit_cny 20` |
| 清领星缓存 | `ivyea lingxing cache clear` |
